# Software Requirements Specification (SRS)
## 드론 항공촬영 임무 평가 시스템

**버전**: 1.0  
**작성일**: 2026-05-26  
**기준 문서**: `소프트웨어공학_프로젝트.docx`

## 1. Introduction

### 1.1 목적

실제 드론 비행은 장소, 기상 조건, 장비 파손, 안전사고 및 운용 비용의 제약이 있다. 따라서 본 시스템은 AirSim에서 생성된 비행 로그, 촬영 로그, 충돌 로그, 이미지 데이터를 입력으로 받아 드론 항공촬영 임무 수행 결과를 평가하는 교육용 실습 플랫폼 구현을 목적으로 한다.

### 1.2 범위

본 시스템은 AirSim 기반 시뮬레이션에서 수행된 드론 항공촬영 임무 결과를 평가하는 것으로 한정한다. 부가 도구로 AirSim 격자 비행 스크립트(`grid_operate.py`)와 데이터 변환 도구(`convert_airsim.py`)를 포함하며, 이를 통해 ODM 정사영상 생성과 연계할 수 있다.

## 2. Overall Description

### 2.1 제품 기능 요약

- 임무 목표 및 평가 기준 설정
- 비행 로그, 촬영 로그, 충돌 로그 불러오기
- 촬영 위치 오차, 방향 오차, 촬영 누락 여부, 충돌 여부 평가
- 평가 점수 및 분석 결과 출력
- 입력 데이터 검증 및 오류 메시지 표시
- AirSim 격자 비행 스크립트를 통한 ODM 호환 데이터셋 생성
- AirSim 데이터셋 변환 도구를 통한 실제 GPS 좌표 포함 로그 생성
- ODM Docker 연동을 통한 정사영상 생성 및 평가 결과 오버레이
- 평가 결과 시각화 및 리포트 저장

### 2.2 사용자 유형

- 드론 조작을 배우고자 하는 학습자
- 드론 교육자
- 시뮬레이션 기반 실습 환경을 검증하는 개발자
- 시뮬레이션 환경 관련 종사자

### 2.3 운영 환경

- 본 시스템은 Windows 기반 일반 Desktop PC 환경에서 실행된다.
- 입력 파일과 출력 파일을 저장하고 불러올 수 있는 로컬 파일 시스템 환경을 요구한다.
- 드론 조작 장치를 직접 입력으로 사용하지 않으며, 외부 시뮬레이션 도구에서 생성된 로그 데이터를 입력으로 사용한다.

### 2.4 제약 조건

| 제약조건 | 세부사항 |
|---|---|
| 데이터 의존성 | 외부 시뮬레이션 도구에서 생성된 데이터를 입력으로 사용하므로, 해당 데이터가 없으면 평가를 수행할 수 없다. |
| 평가 신뢰성 | 평가 결과는 입력 로그와 임무 설정 파일의 정확성에 의존한다. 로그 데이터가 누락되거나 부정확한 경우 평가 결과의 신뢰성이 낮아질 수 있다. |
| 구현 범위 제한 | 드론을 직접 조작하거나 Unity 또는 Unreal Engine 내부 서비스로 구현하지 않고, 로그 데이터를 분석하는 프로그램으로 한정한다. |

### 2.5 가정 및 의존성

- 드론 조작과 가상환경 실행은 AirSim 등 외부 시뮬레이션 도구에서 수행된다고 가정한다.
- 입력 로그와 임무 설정 파일은 사전에 정의된 형식을 따른다고 가정한다.
- 평가 항목별 가중치는 임무 설정 파일에 정의된 값을 사용한다고 가정한다.
- 촬영 로그에는 이미지 파일 경로가 포함되어야 하며, 이미지 파일이 없을 경우 해당 촬영 기록은 이미지 연결 오류로 처리한다.
- 특정 Unreal Engine 버전 또는 Unity/Unreal 내부 구현에 직접 종속되지 않도록 설계한다.

## 3. System Features

### 3.1 주요 기능

#### 입력 및 로드

| 기능 | 세부사항 |
|---|---|
| 임무 설정 로드 | JSON 설정 파일에서 목표 촬영 위치, 목표 촬영 방향, 목표별 제한 시간, 허용 오차, 평가 기준을 불러온다. |
| 로그 수집 | 비행 로그, 촬영 로그, 충돌 로그에서 드론 상태, 촬영 정보, 충돌 정보를 불러온다. |
| 입력 데이터 검증 | 임무 설정 파일, 비행 로그, 촬영 로그, 충돌 로그, 이미지 경로의 누락 여부, 형식 오류, 필수 필드 누락, NaN 값, 중복 목표 ID, 잘못된 가중치 값을 확인한다. |
| 임무 설정 확인 | 불러온 임무 설정 파일의 목표 촬영 위치, 목표 촬영 방향, 목표별 제한 시간, 평가 기준을 화면에 표시한다. |

#### 평가 처리

| 기능 | 세부사항 |
|---|---|
| 촬영 조건 평가 | 촬영 위치, 촬영 방향, 제한 시간 조건을 기준으로 목표별 촬영 성공 여부와 누락 여부를 판정하고, 위치 오차와 방향 오차를 계산한다. 목표별 촬영 성공은 해당 목표에 대해 위치 조건, 방향 조건, 시간 조건이 모두 만족된 경우에만 인정한다. |
| 충돌 정보 평가 | 충돌 로그를 기준으로 비행 중 충돌 발생 여부, 충돌 시간, 충돌 위치를 확인한다. |
| 시간 초과 평가 | 비행 시간과 촬영 시간을 기준으로 각 목표 촬영 지점에 정의된 제한 시간 초과 여부를 판정한다. |
| 점수 산출 | 촬영 성공 여부, 위치 오차, 방향 오차, 충돌 여부, 시간 초과 여부를 종합하여 최종 점수를 산출한다. |
| 진행 상태 표시 | 평가 실행 중 파일 로드, 입력 검증, 평가 계산, 결과 저장의 진행 상태를 화면에 표시한다. |

#### 결과 표시

| 기능 | 세부사항 |
|---|---|
| 감점 세부 내역 표시 | 촬영 위치, 촬영 방향, 촬영 시간 초과, 촬영 누락, 충돌 여부에 대한 항목별 감점과 가중치를 화면에 표시한다. |
| 상세 평가 결과 표시 | 목표 촬영 지점별 위치 오차, 방향 오차, 위치 조건 만족 여부, 방향 조건 만족 여부, 시간 조건 만족 여부, 촬영 성공 여부, 누락 여부, 시간 초과 여부를 표 형태로 표시한다. |
| 촬영 이미지 미리보기 | 촬영 로그와 연결된 이미지 파일을 결과 화면에서 확인할 수 있도록 표시한다. |
| 평가 결과 시각화 | 위치 오차, 방향 오차, 촬영 성공 여부, 충돌 여부, 최종 점수를 그래프 또는 차트 형태로 표시한다. |

#### 저장 및 예외 처리

| 기능 | 세부사항 |
|---|---|
| 리포트 생성 및 결과 저장 | 평가 결과, 감점 세부 내역, 임무별 평가 요약, 목표별 상세 평가 결과를 CSV 또는 JSON 형식으로 저장한다. |
| 오류 메시지 표시 | 입력 파일 누락, 형식 오류, 이미지 경로 오류가 발생한 경우 사용자가 원인을 확인할 수 있도록 오류 메시지를 제공한다. |
| 예외 처리 기준 | 임무 설정 파일 또는 필수 로그 파일이 누락된 경우 평가는 실행하지 않는다. 촬영 이미지 파일이 누락된 경우 해당 촬영 기록은 이미지 연결 오류로 표시한다. 목표 촬영 지점에 대응되는 촬영 로그가 없는 경우 해당 목표는 촬영 누락으로 판정한다. 로그 파일에 빈 줄, 잘못된 형식, NaN 값이 포함된 경우 해당 행은 오류 항목으로 기록하고 평가 대상에서 제외한다. |

### 3.2 평가 기준

#### 입력 검증

| 평가기준 | 세부사항 |
|---|---|
| 입력 데이터 유효성 | 임무 설정 파일과 로그 파일에 평가에 필요한 필수 항목이 포함되어 있는지 확인한다. |
| 이미지 경로 유효성 | 촬영 로그에 기록된 이미지 경로가 실제 이미지 파일과 연결되어 있는지 확인한다. |

#### 촬영 조건 평가

| 평가기준 | 세부사항 |
|---|---|
| 촬영 위치 | 목표 촬영 위치와 실제 촬영 위치 사이의 거리 오차를 계산하고, 허용 위치 오차 이내인지 확인한다. |
| 촬영 방향 | 목표 촬영 방향과 실제 카메라 방향의 Yaw 및 Pitch 각도 오차를 각각 계산하고, 허용 방향 오차 이내인지 확인한다. |
| 촬영 시간 | 비행 로그의 첫 번째 유효 기록 시각을 임무 시작 시각으로 한다. 각 목표 촬영 지점에 대해, 매칭된 촬영 기록의 촬영 시간이 임무 시작 시각으로부터 해당 목표에 정의된 제한 시간 이내인지 확인한다. |
| 촬영 누락 | 목표 촬영 지점별로 매칭된 촬영 기록이 존재하는지 확인한다. 매칭된 촬영 기록이 없는 경우 해당 목표는 촬영 누락으로 판정한다. |
| 목표-촬영 로그 매칭 기준 | 각 목표 촬영 지점은 모든 유효 촬영 기록을 대상으로 1:1 매칭한다. 하나의 촬영 기록은 하나의 목표 촬영 지점에만 매칭될 수 있다. 매칭은 위치 오차와 방향 오차에 임무 설정 파일의 평가 항목별 가중치를 적용한 종합 오차가 최소가 되도록 수행한다. 허용 위치 오차, 허용 방향 오차, 제한 시간 조건 위반 여부는 매칭 후보에서 사전 제외하지 않고, 매칭 이후 각 목표에 대해 개별적으로 판정한다. |

#### 충돌 평가

| 평가기준 | 세부사항 |
|---|---|
| 충돌 여부 | 충돌 로그를 기준으로 비행 중 충돌 발생 여부를 확인하고, 충돌이 발생한 경우 충돌 시간과 충돌 위치를 기록한다. |

#### 점수 평가

| 평가기준 | 세부사항 |
|---|---|
| 목표별 상세 평가 | 각 목표 촬영 지점마다 위치 조건, 방향 조건, 시간 조건, 이미지 연결 여부를 개별적으로 평가한다. 목표별 성공 여부는 위치 조건, 방향 조건, 시간 조건을 모두 만족하는 경우에만 True로 판정한다. |
| 감점 세부 내역 | 촬영 위치, 촬영 방향, 시간 초과, 촬영 누락, 충돌 여부에 대한 항목별 감점과 가중치를 확인한다. |
| 점수 산출 기준 | 최종 점수는 기준 점수 100점에서 평가 항목별 감점을 차감하는 방식으로 산출한다. 위치 오차와 방향 오차는 허용 기준 초과 정도에 따라 감점하며, 촬영 누락, 충돌 발생, 시간 초과는 평가 기준에 따라 감점한다. 감점 누적 결과가 0점 미만이 되는 경우 최종 점수는 0점으로 처리한다. |
| 최종 점수 | 각 평가 항목의 감점 결과를 종합하여 기준 점수 100점에서 차감한 최종 점수를 산출한다. |

## 8. ODM 정사영상 연동 (선택 기능)

### 8.1 목적

OpenDroneMap(ODM)은 항공 이미지를 정사영상 및 3D 모델로 처리하는 오픈소스 사진측량 도구이다. 본 시스템은 ODM을 외부 엔진으로 호출하여 정사영상을 생성하고, 생성된 정사영상 위에 임무 평가 결과를 오버레이하는 기능을 선택적으로 제공한다.

ODM은 AGPLv3 라이선스로 배포되며, 본 시스템은 ODM 소스코드를 포함하지 않고 Docker를 통해 외부 프로세스로 실행한다.

### 8.2 기능 요구사항

| 기능 | 세부사항 |
|---|---|
| 프로젝트 준비 | 촬영 로그의 NED 좌표를 GPS 좌표로 변환하여 ODM geo.txt 파일을 생성하고, 이미지 파일을 ODM 프로젝트 폴더에 복사한다. |
| ODM 실행 | Docker를 통해 OpenDroneMap을 실행하고 실행 로그를 실시간으로 표시한다. |
| 정사영상 로드 | ODM이 생성한 orthophoto.tif 또는 orthophoto.png 파일을 불러온다. |
| 결과 오버레이 | 정사영상 위에 목표 지점, 촬영 위치, 성공/실패 여부를 오버레이하여 표시한다. |

### 8.3 제약 조건

| 제약조건 | 세부사항 |
|---|---|
| Docker 의존성 | 본 기능은 Docker Desktop이 설치되어 실행 중인 환경에서만 동작한다. |
| 처리 시간 | ODM 처리 시간은 이미지 수와 해상도에 따라 수 분에서 수십 분이 소요될 수 있다. |
| 좌표계 | 촬영 로그의 NED 로컬 좌표를 가상 GPS 좌표로 변환하여 사용하므로, 실제 지리 좌표와 일치하지 않는다. |
| 라이선스 | ODM은 AGPLv3 라이선스를 따르며, 본 시스템은 ODM을 외부 프로세스로만 호출한다. |

### 8.4 검증 기준

| 검증 기준 | 세부사항 |
|---|---|
| geo.txt 생성 | 촬영 로그 레코드 수와 geo.txt 항목 수가 일치해야 한다. |
| Docker 실행 | Docker가 없는 환경에서 적절한 오류 메시지가 표시되어야 한다. |
| 정사영상 로드 | ODM 결과 파일을 불러와 화면에 표시할 수 있어야 한다. |
| 오버레이 | 목표 지점과 촬영 위치가 정사영상 위에 올바르게 표시되어야 한다. |

## 9. 데이터 수집 도구 (선택 기능)

### 9.1 격자 비행 스크립트 (grid_operate.py)

| 기능 | 세부사항 |
|---|---|
| 격자 경로 생성 | 중심 좌표, 구역 크기, 고도, 겹침률을 입력받아 지그재그 격자 경로를 생성한다. |
| 수직 촬영 | 카메라를 수직 하향(-90°)으로 설정하여 ODM에 최적화된 나디르 이미지를 촬영한다. |
| 자동 비행 | AirSim Python API를 통해 격자 경로를 자동 비행하며 각 지점에서 안정화 후 촬영한다. |
| 메타 저장 | 기존 operate 스크립트와 동일한 meta JSON 포맷으로 GPS, IMU, 카메라 정보를 저장한다. |

파라미터:

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `--width` | 80m | 조사 구역 폭 (X 방향) |
| `--height` | 80m | 조사 구역 높이 (Y 방향) |
| `--altitude` | 20m | 비행 고도 |
| `--overlap` | 0.75 | 이미지 겹침률 |
| `--speed` | 4.0 m/s | 비행 속도 |

### 9.2 데이터 변환 도구 (convert_airsim.py)

| 기능 | 세부사항 |
|---|---|
| 비행 로그 변환 | multirotor_kinematics 데이터를 flight_log.csv로 변환한다. |
| 촬영 로그 변환 | camera.pose 데이터를 capture_log.csv로 변환한다. |
| 실제 GPS 포함 | meta JSON의 gps.geo_point에서 실제 위도/경도/고도를 추출하여 capture_log.csv에 포함한다. |
| 충돌 로그 생성 | 충돌 데이터가 없는 경우 빈 collision_log.csv를 생성한다. |
| 샘플 미션 생성 | 촬영 기록 기반 샘플 mission.json을 자동 생성한다. |

### 9.3 전체 시연 흐름

```
AirSim 실행
    ↓
grid_operate.py 실행 (격자 비행 + 촬영)
    ↓
convert_airsim.py 실행 (로그 변환 + GPS 포함)
    ↓
평가 시스템에서 파일 선택 및 평가 실행
    ↓
ODM 탭에서 정사영상 생성 (Docker)
    ↓
평가 결과 오버레이 확인
```
